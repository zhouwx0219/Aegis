# 代码结构交接

说明当前 Data Agent System 的主要代码结构、关键模块和实验入口。

## 1. 项目目标

当前项目是一个面向 Data Agent System 的事务处理研究原型。

核心设计是：

- 底层只提供版本化 KV 存储。
- agent runtime 负责事务语义、候选计划、并发控制策略选择、提交协议和失败重试。
- ATCC、传统 CC、混合策略、commit protocol 都作为可插拔模块接入。

当前重点是 ATCC 在 agent-style TPCC/YCSB workload 上的实验。

## 2. 顶层目录

```text
agent/          Python agent runtime、workload、evaluation 代码
core/           C++/pybind 底层版本化 KV 和并发控制内核
docs/           论文、汇报、交接文档
results/        保留的正式实验结果
scripts/        复现实验和汇总结果的脚本
tests/          单元测试和实验入口回归测试
third_party/    外部参考代码
```

## 3. Runtime 核心模块

### 3.1 `agent/runtime/transaction.py`

事务生命周期入口。

主要对象：

- `AgentTransactionManager`
- `AgentTransaction`

职责：

- 注册 KV 对象；
- 创建事务 snapshot；
- 管理 read set / candidates；
- 管理 pre-snapshot locks；
- 调用 commit protocol；
- 记录 trace；
- 接收可插拔的 `operation_policy` 和 `transaction_atcc_policy`。



### 3.2 `agent/runtime/cc_registry.py`

并发控制策略注册表。

职责：

- 注册传统 CC；
- 解析策略名；
- 把 `adaptive-op-strict` 和 `transaction-atcc-strict` 翻译成具体执行策略；
- 生成 pre-snapshot lock plan；
- 将提交反馈写回 ATCC 策略表。

重要策略名：

```text
occ
2pl-nowait
2pl-wait-die
mvcc-full
silo-full
tictoc-full
adaptive-op-strict
transaction-atcc-strict
adaptive-hybrid
```



### 3.3 `agent/runtime/atcc.py`

ATCC 策略表核心。

当前包含两类 ATCC：

- `PhaseAwareATCCModule`：操作级/阶段感知 ATCC 的动作模块。
- `TransactionAwareATCCModule`：事务级 ATCC，使用事务状态表、阶段动作、hot/cold read/write set 和 OCC fast path。

事务级 ATCC 的动作：

```text
occ
lock-hot-writes
lock-hot-read-write
lock-write-set
lock-read-write-set
```



### 3.4 `agent/runtime/adaptive.py`

操作级 ATCC 和旧 adaptive policy 的主要实现。

关键对象：

- `OperationPolicyTable`
- `OperationPolicyProfile`
- `OperationPolicyDecision`
- `OperationPolicyTelemetry`

它负责 per-operation 的 optimistic/pessimistic 判断。

### 3.5 `agent/runtime/hybrid.py`

family-level 自适应选择器。

职责：

- 在任务或任务窗口级别选择整个策略族；
- 例如 low conflict 选择 OCC，hot write 选择 ATCC，read-heavy 选择 MVCC/TicToc。



### 3.6 `agent/runtime/commit_protocol.py`

提交协议实现。

职责：

- 执行 candidate selection；
- 执行 commit；
- 处理传统 CC 结果；
- 处理 prelock scope；
- 处理刷新、重放、重试相关事件。



## 4. Workload 模块



### 4.1 `agent/workloads/base.py`

定义通用 agent workload 抽象：

- `AgentTask`
- `AgentCandidate`
- `AgentOperation`
- `AgentStage`
- `prepare_task_transaction`
- `populate_task_stage`

这是 TPCC/YCSB agent-style workload 的共同基础。

### 4.2 `agent/workloads/ycsb.py`

构造 agent-style YCSB。

对象格式：

```text
ycsb:record:{record}:field:{field}
```

支持：

- read；
- overwrite；
- hotspot_fraction；
- hotspot_access_probability；
- explore/refine/commit 阶段。



### 4.3 `agent/workloads/tpcc.py`

构造 agent-style TPCC。

当前实验核心是 NewOrder。

关键对象：

```text
tpcc:district:{w}:{d}:next_order_id
tpcc:district:{w}:{d}:orders
tpcc:stock:{w}:{item}:quantity
tpcc:stock:{w}:{item}:ytd
```

NewOrder workflow：

```text
explore: 读取前半库存
refine: 读取后半库存
commit: 分配订单号、追加订单、扣减库存、更新库存销售量
```



## 5. Evaluation 模块



### 5.1 `agent/evaluation/atcc_retry_experiment.py`

最重要的实验 runner。

用途：

- 跑 YCSB/TPCC retry-aware ATCC 对比实验；
- 支持 low/medium/high profile；
- 支持 staged execution；
- 支持 background workers；
- 支持传统 CC、操作级 ATCC、事务级 ATCC、adaptive-hybrid；
- 输出 JSON 结果。

常用策略：

```powershell
--strategies occ,2pl-nowait,2pl-wait-die,mvcc-full,silo-full,tictoc-full,adaptive-op-strict,transaction-atcc-strict
```



### 5.2 `agent/evaluation/atcc_manifest_runner.py`

按 JSON manifest 批量跑实验。

### 5.3 `agent/evaluation/atcc_policy_training.py`

训练/导出 operation-level ATCC policy artifact。

### 5.4 `agent/evaluation/atcc_profile_runner.py`

profile-level 训练和评估入口。

## 6. Scripts

当前保留 3 个脚本：

```text
scripts/run_ycsb_compare.ps1
scripts/run_tpcc_compare.ps1
scripts/summarize_retry_results.py
```

用途：

- `run_ycsb_compare.ps1`：复现 YCSB low/medium/high 对比。
- `run_tpcc_compare.ps1`：复现 TPCC low/medium/high 对比。
- `summarize_retry_results.py`：把多个 JSON 汇总成 `summary.csv`。



## 7. Tests

测试目录保留所有 `.py` 回归测试。不要删除这些测试，它们覆盖 core runtime、traditional CC、ATCC、manifest runner、compare scripts 和事务级 ATCC。

本轮新增重点测试：

```text
tests/test_transaction_atcc.py
```

它验证：

- 低风险事务走 OCC fast path；
- hot write commit 阶段只锁 hot write set；
- registry 能生成 `transaction-atcc-strict` 的 pre-snapshot plan；
- `AgentTransactionManager` 能注入事务级 ATCC policy；
- runner 能按 workload 选择 TPCC/YCSB 的事务级 ATCC 表。

推荐回归命令：

```powershell
wsl bash -lc "cd /mnt/z/Data-Agent-System-master/cast-das && python3 -m unittest tests.test_core tests.test_compare_scripts tests.test_atcc_family_policy tests.test_atcc_policy_training tests.test_atcc_profile_runner tests.test_atcc_manifest_runner tests.test_atcc_retry_experiment tests.test_atcc_ycsb_strict_tuned tests.test_traditional_cc_full tests.test_transaction_atcc -v"
```
